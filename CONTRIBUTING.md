# Contributing to BOSC Community Library

First off, thank you for considering contributing to the Bugema Open Source Community! It's people like you that make the BOSC Community Library such a great tool for everyone.

## How Can I Contribute?

### Reporting Bugs

- **Check for existing issues.** Someone might have already reported the bug.
- **Open a new issue.** Use the "Bug Report" template and provide as much detail as possible.

### Suggesting Enhancements

- **Check for existing feature requests.**
- **Open a new issue.** Use the "Feature Request" template and describe the problem you're trying to solve.

### Pull Requests

1.  **Fork the repository.**
2.  **Create a branch.** Use a descriptive name (e.g., `fix/typo-in-readme` or `feat/add-new-resource`).
3.  **Make your changes.**
4.  **Follow the coding standards.**
5.  **Write tests.** (If applicable)
6.  **Submit the PR.** Use the PR template and link any related issues.

## Technical Instructions

- **Documentation:** All changes must be documented in the `docs/` directory.
- **Commit Messages:** Use descriptive commit messages (e.g., `docs: update contributing guide`).
- **Review Process:** All PRs require review from at least one Lead Maintainer before merging.

## Community Standards

Please refer to our [Code of Conduct](CODE_OF_CONDUCT.md) for details on our community standards.
